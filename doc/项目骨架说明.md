## 2. 快速构建

```bash
# 首次配置（自动检测 AVX2；若 CPU 支持 AVX-512 则加 -DAXONFORGE_ENABLE_AVX512=ON）
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release

# 编译（libaxonforge.a + axonforge-cli + 3个测试二进制）
cmake --build build -j$(nproc)

# 运行全部单元测试
ctest --test-dir build -j$(nproc) --output-on-failure
```

### CMake 选项一览

| 选项 | 默认 | 说明 |
|------|------|------|
| `AXONFORGE_BUILD_TESTS` | ON | 构建 Catch2 单元测试 |
| `AXONFORGE_BUILD_CLI` | ON | 构建 `axonforge-cli` |
| `AXONFORGE_ENABLE_AVX2` | ON | 编译时启用 AVX2+FMA（配置时自动验证） |
| `AXONFORGE_ENABLE_AVX512` | OFF | 编译时启用 AVX-512F+VNNI（需手动开启） |
| `AXONFORGE_ENABLE_OPENMP` | ON | OpenMP 多线程 |

## 3. 核心架构决策

### 3.1 Tensor — 视图语义

`Tensor` 是一个**廉价值类型**：只含 `shared_ptr<TensorStorage>` + shape/strides/offset/dtype。  
`reshape / permute / slice` 均为 O(1)，共享底层 `TensorStorage`，**不触发内存分配**。

```cpp
auto t  = Tensor::zeros({4, 8, 64}, DType::F32);   // 分配一次
auto tp = t.permute({1, 0, 2});                     // O(1)，zero-copy
auto ts = t.slice(0, 0, 2);                         // O(1)，zero-copy
auto tc = t.contiguous();                            // 如需连续副本才分配
```

### 3.2 ComputeGraph — 符号维度 IR

- `TensorDesc.shape` 可含 `std::string` 符号维度（如 `"seq_len"`）
- `graph.instantiate({"seq_len", 512})` 绑定具体值后才运行形状推导
- decode 步 seq_len 从 N→1 无需重建图，只需重新 `instantiate`

```cpp
GraphBuilder b(graph);
TensorId x = b.input({int64_t(1), std::string("seq"), int64_t(4096)},
                      DType::F16, "hidden");
```

### 3.3 Backend 插件系统

新增 Backend 步骤（**核心代码零修改**）：

```
1. 创建 src/backend/my_hw/my_backend.cpp
2. 继承 axonforge::IBackend，实现所有纯虚函数
3. 在文件末尾添加注册：
       [[gnu::constructor]] static void register_my_hw() {
           BackendRegistry::instance().register_backend(
               "my_hw", []{ return std::make_unique<MyBackend>(); });
       }
4. 在 src/backend/CMakeLists.txt 中 add_subdirectory(my_hw)
5. 用户：Engine::from_gguf("model.gguf", {.backend = "my_hw"});
```

`[[gnu::constructor]]` 确保注册在 `main()` 之前发生。链接时需配合 `--whole-archive` 防止 linker 死代码消除（CMakeLists 已配置）。

### 3.4 x86 Backend — 运行时 ISA 分级

`cpu_x86/cpuid.cpp` 在首次调用时执行 CPUID，缓存 `CpuFeatures`：

```
Ice Lake+ / Zen4 → AVX-512F + VNNI + BF16  [最优路径]
Skylake-X+ / Zen3 → AVX-512F               [高性能路径]
Haswell+ (2013+) → AVX2 + FMA              [基础路径，当前完整实现]
Legacy → SSE4.2                            [兜底，不建议生产]
```

内核文件通过独立 OBJECT 库编译（`axonforge_kernels_avx2`），接受 `-mavx2 -mfma -O3` 而不污染其他模块的编译选项。

### 3.5 内存分配

| 场景 | 分配器 | 说明 |
|------|--------|------|
| 权重张量（大，持久） | `CpuTensorStorage`（64B对齐 malloc）| mmap 后封装 |
| 前向传播临时张量 | `MemoryArena`（bump-pointer）| 每次 forward 前 reset，O(1) 分配 |
| KV Cache | `ContinuousKvCache` / `PagedKvCache` | 按 max_seq_len 预分配 |
