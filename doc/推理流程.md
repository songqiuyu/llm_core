## 阶段一：模型加载 `Engine::from_gguf()`

**文件**：`src/engine.cpp:59`

```
GGUF 文件（mmap）
    │
    ├─ KV 元数据解析
    │     general.architecture  → "gpt2"
    │     gpt2.block_count      → n_layers = 12
    │     gpt2.embedding_length → n_embd   = 768
    │     gpt2.attention.head_count → n_heads = 12
    │     gpt2.context_length   → n_ctx    = 1024
    │     tokenizer.ggml.bos_token_id → 50256（GPT-2 无 BOS，实际不用）
    │     tokenizer.ggml.eos_token_id → 50256
    │
    ├─ 词表加载
    │     tokenizer.ggml.tokens → vocabulary[50257]（raw bytes 存储）
    │
    └─ 权重哈希表（零拷贝）
          对每个张量 i：
            name  = "model/wte", "model/h0/attn/c_attn/w", ...
            Tensor = mmap 地址的 raw view（不拷贝数据）
          结果：unordered_map<string, Tensor>，148 个张量
```

**关键设计**：`GgufReader` 用 `mmap` 映射整个 GGUF 文件（239 MB），`Tensor::from_raw_blob()` 直接持有 mmap 指针，全程零拷贝。

## 阶段二：分词 `gpt2_encode_simple()`

**文件**：`src/models/gpt2_model.cpp:509`

```
输入：std::string_view text = "Hello, I am"

1. 构建反向查找表（一次性）
   vocabulary[id] → string_view，填入 unordered_map<string_view, id>

2. 贪心最长匹配（从左到右）
   pos = 0
   while pos < len(text):
       尝试 text[pos : pos+50], text[pos : pos+49], ...
       找到最长匹配的词表项 → push token_id
       pos += match_len

输出：token_ids = [15496, 11, 314, 716]   （"Hello", ",", " I", " am"）
```

> **注意**：这是简化实现，不含 BPE merge rules，对空格处理依赖词表中存储了 `" am"` 这样的带空格 token。完整 BPE tokenizer 是 Phase 2 目标。

## 阶段四：Prefill（处理 prompt）

每个 prompt token 按顺序调用 `gpt2_forward_token(token_id, pos=i, state)`：

```
for i in 0 .. len(prompt_ids)-1:
    gpt2_forward_token(prompt_ids[i], pos=i, state)
    state.n_past++
```

Prefill 和 Decode 共享同一个 forward 函数，区别仅在于：
- Prefill：`pos` 从 0 递增，每次 attention 只看 `[0, pos]`
- Decode：`pos = n_past`，KV cache 已含所有历史

## 阶段五：单 Token 前向传播 `gpt2_forward_token()`

**文件**：`src/models/gpt2_model.cpp:248`

### 5.1 词嵌入

```
wte: F16[50257 × 768]  — token embedding（mmap 只读）
wpe: F32[1024  × 768]  — position embedding

x[768] = F16→F32(wte[token_id]) + wpe[pos]
```

张量形状：`wte` 行是 `token_id`，列是 `embedding_dim`。

### 5.2 Transformer Block × 12 层

每层顺序如下（`l` 从 0 到 11）：

```
┌─ Pre-Attn LayerNorm ──────────────────────────────────────┐
│  xb = x（复制）                                           │
│  xb = LayerNorm(xb, ln1_g[768], ln1_b[768])              │
│  公式：xb_i = (xb_i - mean) / sqrt(var + 1e-5) * g_i + b_i │
└───────────────────────────────────────────────────────────┘
         ↓
┌─ QKV 投影（最热 GEMV，F16 权重）──────────────────────────┐
│  qkv[2304] = c_attn_w[2304×768] × xb[768] + c_attn_b    │
│  → Q = qkv[0:768]                                        │
│  → K = qkv[768:1536]  → 写入 kv_k[layer][pos][:]        │
│  → V = qkv[1536:2304] → 写入 kv_v[layer][pos][:]        │
│                                                           │
│  执行路径：gemv_f16_dispatch()                            │
│    out=2304 >= 128 && pool != nullptr                    │
│    → pool.parallel_for(2304, gemv_f16_avx2_range)        │
│       8 线程各算 288 行，并发读权重                        │
└───────────────────────────────────────────────────────────┘
         ↓
┌─ 因果多头注意力（标量，seq 通常很短）────────────────────────┐
│  for head h in 0..11:                                     │
│    Q_h = Q[h*64 : h*64+64]                               │
│    for t in 0..pos:                                       │
│      score[t] = dot(Q_h, K_cache[layer][t][h*64:]) / √64 │
│    softmax(score[0..pos])                                 │
│    attn_out[h*64:] += Σ_t score[t] * V_cache[t][h*64:]   │
│                                                           │
│  [注] 单 token decode 时 seq 长度 ≤ 1024，此处标量实现     │
│       是 Phase 2 的优化目标（FlashAttention / AVX2 化）    │
└───────────────────────────────────────────────────────────┘
         ↓
┌─ 注意力输出投影 + 残差─────────────────────────────────────┐
│  xb[768] = c_proj_w[768×768] × attn_out[768] + c_proj_b  │
│  x += xb   （残差连接 1）                                 │
└───────────────────────────────────────────────────────────┘
         ↓
┌─ Pre-FFN LayerNorm ────────────────────────────────────────┐
│  xb = LayerNorm(x, ln2_g, ln2_b)                          │
└───────────────────────────────────────────────────────────┘
         ↓
┌─ FFN up-projection（最热 GEMV）────────────────────────────┐
│  ffn[3072] = c_fc_w[3072×768] × xb[768] + c_fc_b         │
│  ffn = GELU(ffn)                                          │
│  GELU(x) = 0.5x(1 + tanh(√(2/π)(x + 0.044715x³)))       │
└───────────────────────────────────────────────────────────┘
         ↓
┌─ FFN down-projection + 残差───────────────────────────────┐
│  xb[768] = c_proj_w[768×3072] × ffn[3072] + c_proj_b     │
│  x += xb   （残差连接 2）                                 │
└───────────────────────────────────────────────────────────┘
```

### 5.3 Final LayerNorm + LM Head

```
x = LayerNorm(x, lnf_g[768], lnf_b[768])

logits[50257] = wte[50257×768] × x[768]    （权重共享）
  → 同样走 gemv_f16_dispatch()
  → 8 线程并行计算 50257 行，这是最大的单次 GEMV（~77 MB 读取）
```

## 阶段六：采样 `sample_top_k()`

**文件**：`src/models/gpt2_model.cpp:332`

```
输入：logits[50257], temperature=0.8, top_k=40

1. Temperature scaling：scored[i] = logits[i] / temperature

2. Partial sort：找出 top-40 的 (score, token_id) 对
   std::partial_sort → O(V log k) ≈ O(50257 × log40)

3. Softmax over top-40：
   probs[i] = exp(scored[i] - max) / Σ

4. 按概率分布采样（categorical）：
   uniform_real r ∈ [0,1)
   遍历 probs 累加，r < cumsum 时取该 token

输出：next_token_id（例如 2925 = " sorry"）
```

## 阶段七：Decode 循环

**文件**：`src/models/gpt2_model.cpp:416`

```
next_id = sample_top_k(last_logits)   ← 使用 prefill 最后一步的 logits

for step in 0 .. max_new_tokens-1:
    generated.push(next_id)

    ── 流式输出（on_token 回调）──────────────────────
    cfg.on_token(next_id)
      → gpt2_decode_tokens(engine, {next_id})
          vocab[next_id] → raw bytes → 直接 printf + fflush
    ────────────────────────────────────────────────

    if next_id == eos_id (50256): break
    if n_past >= 1024: break

    gpt2_forward_token(next_id, pos=n_past, state)
    n_past++
    next_id = sample_top_k(state.logits)
```

每次迭代耗时（稳定态，n_past ~ 5）：
- 4× GEMV F16（QKV, c_proj, c_fc, c_proj_ffn）× 12 层 = 48 次 GEMV
- 1× LM head GEMV（wte.T）
- 注意力计算（seq 较短时可忽略）
- 总计：~13.5 ms/token @73 tok/s

**文件**：`src/models/gpt2_model.cpp:494`


## 关键数据流与内存布局

```
mmap 区域（只读，239 MB）
  ├── wte  F16[50257×768]   = 77.2 MB   ← LM head 也用这个（weight tying）
  ├── wpe  F32[1024×768]    = 3.1 MB
  └── 12 × layer weights
       c_attn_w  F16[2304×768] = 3.5 MB  ← 最频繁读取的权重之一
       c_fc_w    F16[3072×768] = 4.7 MB
       c_proj_w  F16[768×3072] = 4.7 MB
       c_proj_w  F16[768×768]  = 1.2 MB
       ...

CPU DRAM（读写，~75 MB）
  ├── kv_k  F32[12×1024×768] = 37.7 MB  ← decode 时按 pos 增量写入
  ├── kv_v  F32[12×1024×768] = 37.7 MB
  └── scratch buffers        ~ 0.8 MB
```

## GEMV 分发路径

所有 GEMV 调用统一经过 `gemv_f16_dispatch()`：

```
gemv_f16_dispatch(y, W_f16, x_f32, b_f32, out, in, pool)
        │
        ├─ out < 128 或 pool == nullptr
        │     → kGemvRange(y, W, x, b, 0, out, in)   [单线程]
        │
        └─ out >= 128 && pool != nullptr
              → pool.parallel_for(out, λ(s, e) {
                    kGemvRange(y, W, x, b, s, e, in)  [8线程并行]
                })

kGemvRange（进程启动时选择一次）：
  cpu_features().avx2 == true
      → gemv_f16_avx2_range()   [AVX2 + F16C + FMA]
  else
      → gemv_f16_scalar_range()  [通用 fallback]
```

`gemv_f16_avx2_range` 内部（以 QKV 投影为例，out=2304, in=768）：

```
Thread 0 负责 rows [0, 288)，Thread 1 负责 [288, 576)，...

每行（768 F16 权重）：
  for k in 0, 16, 32, ..., 768:
    w0 = _mm256_cvtph_ps(W[k:k+8])    // 8×F16 → 8×F32，1 cycle
    w1 = _mm256_cvtph_ps(W[k+8:k+16])
    acc0 = _mm256_fmadd_ps(w0, x[k:k+8],     acc0)  // FMA，0.5 cycle
    acc1 = _mm256_fmadd_ps(w1, x[k+8:k+16],  acc1)
  y[row] = hsum256(acc0 + acc1) + b[row]
```
