# AxonForge 3B 级模型扩展计划与验证

> 时间：2026-05-13  
> 目标：优先覆盖 4GB 左右端侧 CPU 可运行的 3B/4B 开源模型。

## 一、推荐模型矩阵

| 模型 | GGUF arch | Q4_K_M 大小 | 当前策略 | 竞争力 |
|------|-----------|-------------|----------|--------|
| Qwen2.5-3B-Instruct | `qwen2` | ~2.1 GB | 复用 Qwen2 路径 | 通用中文/英文、小端侧主力 |
| Qwen2.5-Coder-3B-Instruct | `qwen2` | ~2.1 GB | 复用 Qwen2 路径 | 代码生成与工程场景 |
| Qwen3-4B | `qwen3` | ~2.5 GB | 实验支持 dense Qwen3 | reasoning、agent、多语种 |
| DeepSeek-R1-Distill-Qwen-1.5B | `qwen2` | ~1.1 GB | 已支持 | 小 reasoning baseline |
| Llama-3.2-3B-Instruct | `llama` | ~2 GB Q4 | 后续验证 tokenizer/rope | 横向生态对比 |
| SmolLM3-3B | `smollm3` | ~1.98 GB | 后续新增 arch | Apache 2.0、长上下文 |

DeepSeek R1 distill 主线在小模型上是 1.5B，然后跳到 7B/8B；没有优先级足够高的官方 3B 档，因此本轮不作为 3B 主目标。

## 二、本轮实现

- Qwen2.5-3B / Qwen2.5-Coder-3B：继续使用 `qwen2` 分发，复用 Qwen BPE、QKV bias、Q4_K_M/Q6_K/F16/Q8_0 权重路径。
- Qwen3-4B：新增 `qwen3` 分发到 Qwen tokenizer/generation 路径，并在共享 LLaMA-like forward 中支持可选 `blk.N.attn_q_norm.weight`、`blk.N.attn_k_norm.weight`。
- Qwen3 的 Q/K norm 在 Q/K projection 和 bias 之后、RoPE 之前按 head_dim 独立 RMSNorm；Qwen2/LLaMA 模型没有这些 tensor 时保持原行为。
- 上下文仍默认 cap 到 4096，避免 32K/128K metadata 在端侧直接触发过大的 KV cache 分配。

## 三、下载与 Smoke 命令

```bash
mkdir -p models/qwen2.5-3B
hf download Qwen/Qwen2.5-3B-Instruct-GGUF \
    qwen2.5-3b-instruct-q4_k_m.gguf \
    --local-dir models/qwen2.5-3B

mkdir -p models/qwen2.5-coder-3B
hf download Qwen/Qwen2.5-Coder-3B-Instruct-GGUF \
    qwen2.5-coder-3b-instruct-q4_k_m.gguf \
    --local-dir models/qwen2.5-coder-3B

mkdir -p models/qwen3-4B
hf download Qwen/Qwen3-4B-GGUF \
    Qwen3-4B-Q4_K_M.gguf \
    --local-dir models/qwen3-4B
```

```bash
build/tools/cli/axonforge-cli \
    -m models/qwen2.5-3B/qwen2.5-3b-instruct-q4_k_m.gguf \
    -p "The capital of France is" -n 64 -t 0.0 -V

build/tools/cli/axonforge-cli \
    -m models/qwen2.5-coder-3B/qwen2.5-coder-3b-instruct-q4_k_m.gguf \
    -p "Write a Python function that returns the nth Fibonacci number." -n 96 -t 0.0 -V

build/tools/cli/axonforge-cli \
    -m models/qwen3-4B/Qwen3-4B-Q4_K_M.gguf \
    -p "What is 2+2? /no_think" -n 64 -t 0.7 --top-k 20 --top-p 0.8 --rep-penalty 1.5 -V
```

## 四、公平 Benchmark 规则

- AxonForge 和 llama.cpp 使用同一 GGUF、同一 prompt、同一线程数、同一 context cap、同一生成 token 数。
- 单次 smoke 只记录“能否合理生成”；性能结论必须使用固定 benchmark 命令重复运行。
- 建议记录：TTFT、prefill tok/s、decode tok/s、总时间、peak RSS。
- Qwen3 thinking 模式不要用 greedy 作为唯一结论；官方建议 thinking/non-thinking 使用不同采样参数。

## 五、已知限制

- Qwen3 仅覆盖 dense Qwen3；不支持 Qwen3-MoE、Qwen3-Next、Qwen3-VL、多模态 projector。
- Qwen3 tokenizer 目前沿用 Qwen BPE 路径；如果后续做 logits/token ID 严格对齐，需要增加与 llama.cpp 的随机文本 tokenizer 对齐测试。
- Llama-3.2-3B、SmolLM3-3B、Gemma 3、Phi-4-mini 尚未纳入本轮实现，建议作为下一轮横向覆盖任务。
